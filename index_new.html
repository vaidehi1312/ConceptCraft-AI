Retrieval Pipeline
Explainability & Analysis

A transparent view into how ConceptCraftAI finds and validates 3D models. Every query flows through a multi-stage pipeline with dual-track retrieval.

How Models Are Retrieved

0
Query Expansion

Short queries like "heart" or "saturn" are expanded locally with domain context (e.g. "human heart cardiac anatomy organ") to improve recall. No API calls.

1
Dual-Track Retrieval

Two sources searched in parallel:

FAISS Semantic search across 4 local vector databases (biological, physical, chemical, astronomical)
Sketchfab Live API search for high-quality embeddable 3D models
2
CrossEncoder Reranking

Each track is independently rescored using a CrossEncoder (ms-marco-MiniLM-L-6-v2) against the original query for precision.

3
Scoring & Name-Match Boost

Final score = CE relevance (55%) + structural richness (20%) + name-match boost (25%). Models whose name contains the query get boosted.

4
Display-Priority Ordering

Sketchfab results always appear first (interactive embeds), then FAISS inline-renderable results, then external-link-only results at the bottom.

bolt
Fallback: TripoSR Generation

Triggered only when both Sketchfab and FAISS return no usable results. Generates a 3D model from a Wikipedia image of the concept using TripoSR. Since this uses a single reference image, the generated model may be less accurate or incomplete compared to curated models.
